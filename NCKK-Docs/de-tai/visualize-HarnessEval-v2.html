2. Taxonomy 3 Chieu — 7 Metrics

D1. Tool Dispatch Efficiency

"Harness goi dung tool khong? Co thua khong?"

M1.1 Correct Selection Rate — % tool calls chon dung (tu TAP HOP acceptable, khong phai 1 ground-truth)
M1.2 Redundant Call Rate — % tool calls khong duoc dung trong 3 buoc sau
M1.3 Tool Utilization Breadth — so tool types dung / tong tools co
D2. Context Utilization

"Nen context mat thong tin khong?"

M2.1 Info Retention Score — BERTScore giua full-context va compacted response
M2.2 Effective Token Ratio — % tokens lien quan (LLM classifier + human validate)
D3. Backend Portability

"Doi LLM khac thi harness con tot khong?"

M3.1 Cross-Backend Std Dev — std(RR) qua Claude, GPT, DeepSeek
M3.2 Min/Max Ratio — min(RR) / max(RR)
Da bo (future work): D4 Safety Enforcement, D5 Session Continuity
Ly do: Giam scope cho kha thi 6 thang. Safety → de tai rieng (SecureCodeBench)

5. 4 Gia thuyet nghien cuu

H1: Tool > Context

Tool system co effect size lon nhat (Cohen's d > 0.5) trong 3 component

Ly do: Tool quyet dinh agent CO THE lam gi

H2: Context co anh huong

Context management co effect size trung binh (Cohen's d 0.3-0.5)

Ly do: Context giup agent HIEU bai toan

H3: Harness matters

Harness config giai thich >= 10% variance (eta-squared)

Da ha tu 20% (v2 goc) xuong 10% theo goi y hoi dong

H4: Harness giam gap

Harness tot giam variance giua cac LLM backends (interaction effect)

Insight gia tri nhat: harness tot "san phang" su khac biet giua models

Nguyen tac: Khong du doan con so cu the. Chi neu HUONG ky vong. Du thanh cong hay bac bo hypotheses deu la dong gop khoa hoc.

7. 3 Dieu kien thong qua (can xu ly)

DK1: Fix thu tu timeline

Pilot study phai SAU khi fork SWE-Agent. Chia pilot thanh 2 pha:
Pilot A (tren SWE-Agent goc) → validate pipeline + cost
Pilot B (tren modular harness) → validate metrics + modularity

DK2: Chinh ANOVA design

Doi tu ANOVA 2 chieu (9x3) sang ANOVA 3 chieu (3 Tool x 3 Context x 3 Backend).
Them GLMM nhu robustness check cho binary outcome (resolve = 0/1).
Report ca ANOVA (de doc) va GLMM (chinh xac).

DK3: Ha H3 threshold

Doi eta-squared >= 0.20 (large) → >= 0.10 (medium).
Hoac bo threshold, chi report observed eta-squared.
Ly do: 0.20 qua cao, 0.15 van la phat hien co y nghia.

8. 5 Cau hoi phan bien vong 2 (moi)

1Pilot (Tuan 7-8) xay ra TRUOC fork (Tuan 9-11). Pilot chay tren harness nao?
Tra loi: Chia pilot 2 pha. Pilot A tren SWE-Agent goc (validate pipeline). Pilot B tren modular harness (validate metrics).
2Resolve rate la binary (0/1). ANOVA gia dinh normal distribution. Xu ly sao?
Tra loi: Dung ca ANOVA (de trinh bay) va GLMM (chinh xac). Kiem tra ket luan nhat quan. CLT voi n=150 cho phep aggregate.
3V2 dung Claude Haiku classifier thay GPT-judge. Van la LLM danh gia LLM?
Tra loi: Haiku lam classification (don gian), khong phai evaluation. Khac model family voi agent. Validate accuracy tren 200 segments voi human. Chap nhan neu >90%.
4SWE-Agent update lien tuc. Fork bi outdated 2-3 thang. Ket qua con valid?
Tra loi: Do RELATIVE contribution, khong phai absolute. Ghi commit hash. Acknowledge trong Discussion la limitation chung cua open-source research.
5H4 voi chi 3 backends — variance tren 3 diem rat khong on dinh?
Tra loi: Dung interaction term trong ANOVA thay vi tinh std truc tiep. Interaction plot: neu cac duong hoi tu o harness tot → H4 duoc support.

9. Threats to Validity & Risk Analysis

Internal Validity

Confounding: Tat tool thay doi bai toan, khong chi harness. → Giam tools theo muc do, khong tat hoan toan

Non-determinism: LLM output khac moi lan. → temperature=0, 3 runs cho conditions chinh

Docker instability: Mot so env fail. → Pre-filter tasks, retry 2 lan

External Validity

Chi Python: Tool system Python (pytest, pip) khac Java (maven). → Caution khi generalize

Chi open-source: Claude Code, Cursor co the khac. → Future work

Survivor bias: Tasks bi loai (Docker fail) co the la tasks kho. → Report tasks bi loai

Construct Validity

M1.1: Phu thuoc annotator. → 2 annotators, Fleiss' kappa

M2.2: LLM classifier. → Validate 200 segments voi human

LLM versioning: Model bi update trong 6 thang. → Ghi version, chay nhanh

6 Risks & Mitigation

1. Docker fail nhieu → Pre-filter + backup tasks

2. API cost vuot → Pilot uoc tinh truoc

3. Fork kem >3% → Giu nguyen code, chi refactor config

4. Metrics khong phan biet → Pilot phat hien som

5. Expert khong co thoi gian → Online survey

6. DeepSeek API unstable → Backup: Qwen/Llama
