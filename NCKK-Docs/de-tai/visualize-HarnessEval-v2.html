2. Do gi? 5 Chieu danh gia Harness (Day du)

D1: Tool Dispatch Efficiency V2 CHINH

"Harness goi dung tool khong? Co bi thua khong?"

M1.1: Correct Tool Selection Rate
Do gi: % tool calls chon dung tool cho muc dich
Vi du: Agent can doc file → goi Read (dung) hay goi Bash cat (sai tool)?
Cach do: 2 annotators gan nhan "acceptable/not" tren 500 tool calls mau. Dung TAP HOP acceptable tools (nhieu cach dung).
Tu dong? Khong — can nguoi. Validate bang Fleiss' kappa >= 0.6
M1.2: Redundant Call Rate
Do gi: % tool calls ma ket qua KHONG duoc su dung trong 3 buoc tiep theo
Vi du: Agent goi Grep "error" 5 lan cho cung 1 tu → 4 lan la thua
Cach do: Tu dong — log analysis: tool output co xuat hien trong context 3 turns sau khong?
Tu dong? Co. Validate tren 100 mau voi human check
M1.3: Tool Utilization Breadth
Do gi: So loai tool duoc su dung / tong so tools co san
Vi du: Harness co 12 tools nhung agent chi dung 4 → breadth = 33% (thap = khong tan dung)
Cach do: Tu dong — dem unique tool types trong log
Tu dong? Co
D2: Context Utilization V2 CHINH

"Harness nen context tot khong? Co mat thong tin quan trong?"

M2.1: Info Retention Score
Do gi: % thong tin quan trong con lai sau khi nen context
Vi du: Context 100K tokens, sau compaction con 30K. Nhung mat thong tin ve file quan trong → Info Retention thap
Cach do: Chay cung task 2 lan (full context vs compacted). Tinh BERTScore giua 2 outputs
Tu dong? Co. Validate voi human preference tren 30 cap
M2.2: Effective Token Ratio
Do gi: % tokens trong context thuc su lien quan den task hien tai
Vi du: Context 50K tokens nhung chi 15K lien quan → ratio = 30% (70% la "rac")
Cach do: LLM classifier (Claude Haiku) phan loai tung segment: relevant/irrelevant. Validate accuracy tren 200 segments voi human labels
Tu dong? Ban tu dong. Chap nhan neu classifier accuracy >90%
D3: Safety Enforcement FUTURE WORK

"Harness chan duoc lenh nguy hiem khong?"

M3.1: Destructive Op Block Rate
Do gi: % lenh nguy hiem (rm -rf, git push --force) bi chan
Vi du: Agent bi trick goi rm -rf / → harness chan duoc? Inject 50 destructive commands de test
M3.2: Injection Resist Rate
Do gi: % prompt injection bi phat hien va chan
Vi du: File .cursorrules chua lenh doc hai → harness phat hien? (Lien quan vu Claude Code leak)
Tai sao bo khoi v2? Giam scope cho 6 thang. Safety se la de tai rieng: SecureCodeBench (Gap 2).
D4: Session Continuity FUTURE WORK

"Harness co suy giam theo thoi gian khong?"

M4.1: Performance Decay
Do gi: % giam hieu suat sau 100+ turns
Vi du: Turn 10: agent lam dung yeu cau. Turn 150: agent "quen" yeu cau ban dau. Day goi la "instruction fade-out" — harness tot se chen reminders de chong [A1]
M4.2: Instruction Adherence
Do gi: % responses van tuan thu system prompt goc
Vi du: System prompt noi "chi sua bug, khong refactor" nhung turn 100 agent bat dau refactor → adherence giam
Tai sao bo khoi v2? Can long-running experiments (200+ turns) ton nhieu thoi gian va chi phi. De cho future work.
D5: Backend Portability V2 CHINH

"Doi model khac thi harness con tot khong?"

M5.1: Cross-Backend Std Dev
Do gi: Do lech chuan cua resolve rate khi doi LLM backend
Vi du:
Harness A + Claude: 70%, GPT: 65%, DeepSeek: 60% → std = 5.0% (tot, portable)
Harness B + Claude: 80%, GPT: 30%, DeepSeek: 20% → std = 32.1% (te, chi tot cho Claude)
Cach do: std(RR_claude, RR_gpt, RR_deepseek). Std thap = portable
Tu dong? Co
M5.2: Min/Max Ratio
Do gi: Ty le performance backend yeu nhat / backend manh nhat
Vi du:
Harness A: min=60%, max=70% → ratio = 0.86 (tot)
Harness B: min=20%, max=80% → ratio = 0.25 (te)
Cach do: min(RR) / max(RR). Gan 1.0 = portable. Gan 0 = phu thuoc 1 model
Tu dong? Co

5. 4 Gia thuyet nghien cuu

H1: Tool > Context

Tool system co effect size lon nhat (Cohen's d > 0.5) trong 3 component

Ly do: Tool quyet dinh agent CO THE lam gi

H2: Context co anh huong

Context management co effect size trung binh (Cohen's d 0.3-0.5)

Ly do: Context giup agent HIEU bai toan

H3: Harness matters

Harness config giai thich >= 10% variance (eta-squared)

Da ha tu 20% (v2 goc) xuong 10% theo goi y hoi dong

H4: Harness giam gap

Harness tot giam variance giua cac LLM backends (interaction effect)

Insight gia tri nhat: harness tot "san phang" su khac biet giua models

Nguyen tac: Khong du doan con so cu the. Chi neu HUONG ky vong. Du thanh cong hay bac bo hypotheses deu la dong gop khoa hoc.

7. 3 Dieu kien thong qua (can xu ly)

DK1: Fix thu tu timeline

Pilot study phai SAU khi fork SWE-Agent. Chia pilot thanh 2 pha:
Pilot A (tren SWE-Agent goc) → validate pipeline + cost
Pilot B (tren modular harness) → validate metrics + modularity

DK2: Chinh ANOVA design

Doi tu ANOVA 2 chieu (9x3) sang ANOVA 3 chieu (3 Tool x 3 Context x 3 Backend).
Them GLMM nhu robustness check cho binary outcome (resolve = 0/1).
Report ca ANOVA (de doc) va GLMM (chinh xac).

DK3: Ha H3 threshold

Doi eta-squared >= 0.20 (large) → >= 0.10 (medium).
Hoac bo threshold, chi report observed eta-squared.
Ly do: 0.20 qua cao, 0.15 van la phat hien co y nghia.

8. 5 Cau hoi phan bien vong 2 (moi)

1Pilot (Tuan 7-8) xay ra TRUOC fork (Tuan 9-11). Pilot chay tren harness nao?
Tra loi: Chia pilot 2 pha. Pilot A tren SWE-Agent goc (validate pipeline). Pilot B tren modular harness (validate metrics).
2Resolve rate la binary (0/1). ANOVA gia dinh normal distribution. Xu ly sao?
Tra loi: Dung ca ANOVA (de trinh bay) va GLMM (chinh xac). Kiem tra ket luan nhat quan. CLT voi n=150 cho phep aggregate.
3V2 dung Claude Haiku classifier thay GPT-judge. Van la LLM danh gia LLM?
Tra loi: Haiku lam classification (don gian), khong phai evaluation. Khac model family voi agent. Validate accuracy tren 200 segments voi human. Chap nhan neu >90%.
4SWE-Agent update lien tuc. Fork bi outdated 2-3 thang. Ket qua con valid?
Tra loi: Do RELATIVE contribution, khong phai absolute. Ghi commit hash. Acknowledge trong Discussion la limitation chung cua open-source research.
5H4 voi chi 3 backends — variance tren 3 diem rat khong on dinh?
Tra loi: Dung interaction term trong ANOVA thay vi tinh std truc tiep. Interaction plot: neu cac duong hoi tu o harness tot → H4 duoc support.

9. Threats to Validity & Risk Analysis

Internal Validity

Confounding: Tat tool thay doi bai toan, khong chi harness. → Giam tools theo muc do, khong tat hoan toan

Non-determinism: LLM output khac moi lan. → temperature=0, 3 runs cho conditions chinh

Docker instability: Mot so env fail. → Pre-filter tasks, retry 2 lan

External Validity

Chi Python: Tool system Python (pytest, pip) khac Java (maven). → Caution khi generalize

Chi open-source: Claude Code, Cursor co the khac. → Future work

Survivor bias: Tasks bi loai (Docker fail) co the la tasks kho. → Report tasks bi loai

Construct Validity

M1.1: Phu thuoc annotator. → 2 annotators, Fleiss' kappa

M2.2: LLM classifier. → Validate 200 segments voi human

LLM versioning: Model bi update trong 6 thang. → Ghi version, chay nhanh

6 Risks & Mitigation

1. Docker fail nhieu → Pre-filter + backup tasks

2. API cost vuot → Pilot uoc tinh truoc

3. Fork kem >3% → Giu nguyen code, chi refactor config

4. Metrics khong phan biet → Pilot phat hien som

5. Expert khong co thoi gian → Online survey

6. DeepSeek API unstable → Backup: Qwen/Llama
