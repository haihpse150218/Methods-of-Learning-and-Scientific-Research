# [2026] Loose — Coverage-Guided Multi-Agent Harness Generation for Java Library Fuzzing

## Metadata
- **arXiv:** 2603.08616v1
- **Venue:** (preprint, arXiv cs.SE)
- **Year:** 2026
- **Authors:** Nils Loose, Nico Winkel, Kristoffer Hempel, Felix Machtle, Julian Hans, Thomas Eisenbarth (University of Lubeck, Institute for IT Security)

## 1. Van de (Problem)
Coverage-guided fuzzing of Java libraries requires specialized fuzz harnesses that translate fuzzer-generated bytes into valid API invocations. Manual harness creation is time-consuming, requiring deep understanding of API semantics, initialization sequences, and exception handling contracts. This limits the number of library APIs that receive comprehensive fuzzing coverage.

## 2. Dong co / Gap
Existing automated harness generation approaches each have complementary limitations: usage-based methods require consumer code corpora that may be unavailable; structure-based methods rely on domain-specific heuristics; feedback-driven methods apply fixed termination thresholds without semantic interpretation of coverage gaps. Recent LLM-based systems either preprocess entire API surfaces (risking context saturation) or lack mechanisms for iterative, query-driven exploration during generation. No existing approach combines LLM reasoning with targeted program analysis, on-demand information retrieval, and semantic interpretation of coverage gaps.

## 3. Phuong phap (Method)
A multi-agent architecture with five specialized ReAct agents orchestrated via LangGraph, using Claude 4.5 Sonnet as the underlying model:
1. **Research agent (RSH):** Explores API documentation, source code, and callgraph via MCP tools to understand target method semantics; produces a structured research report.
2. **Generation agent (GEN):** Synthesizes initial harness source code from the research report, handling input parsing, object construction, and exception handling.
3. **Patching agent (PAT):** Compilation repair -- iteratively diagnoses and fixes build errors by querying source code and documentation.
4. **Coverage analysis agent (CVA):** Interprets coverage gaps by examining uncovered source code via callgraph tools; makes semantic termination decisions (continue vs. stop refinement).
5. **Refinement agent (REF):** Modifies harnesses to address identified coverage deficiencies (diversifying inputs, alternative API paths, edge-case exception handlers).

Key technical innovations:
- **Model Context Protocol (MCP):** Three MCP servers expose Javadoc documentation, source code (via GNU Global), and callgraph information (via SootUp) on demand, preventing context saturation.
- **Method-targeted coverage instrumentation:** Extends JaCoCo to toggle coverage recording only during target method execution, ensuring metrics reflect target behavior rather than framework initialization.
- **Agent-guided termination:** Coverage analysis agent semantically interprets coverage gaps to distinguish addressable deficiencies from fundamental limitations, replacing fixed numerical thresholds.

## 4. Dong gop chinh (Contributions)
- A multi-agent architecture integrating LLM reasoning with program analysis for automated Java fuzz harness generation without requiring consumer code corpora
- A query-driven tool interface using MCP that retrieves precisely scoped program information on demand, preventing context saturation
- Method-targeted coverage instrumentation with agent-guided termination that interprets coverage gaps through source code analysis rather than fixed thresholds
- Empirical validation discovering 3 previously unreported bugs in mature, widely-deployed Java libraries during 12-hour fuzzing campaigns

## 5. Diem manh (Strengths)
- Practical and cost-effective: average $3.20 and 10 minutes per harness, suitable for integration into continuous fuzzing workflows
- Median 26% improvement in method-targeted coverage over OSS-Fuzz baselines; outperforms both OSS-Fuzz and Jazzer AutoFuzz on full package-scope coverage
- Found 3 real bugs in production libraries (commons-cli, jsoup) with zero false positives, validating real-world effectiveness
- Clean separation of concerns across 5 agents with role-based tool access control prevents exploration drift
- On-demand information retrieval via MCP avoids preprocessing entire codebases, enabling scalability to large library ecosystems (115,000+ Maven dependents)

## 6. Han che (Limitations)
- Evaluated on only 7 target methods from 6 libraries; limited generalizability across the broader Java ecosystem
- Does not outperform OSS-Fuzz baseline on jackson-databind where the existing harness contains additional post-target fuzzing logic
- Focused exclusively on Java; applicability to other languages with different fuzzing infrastructure is unexplored
- Single LLM backend (Claude 4.5 Sonnet); no comparison across different models
- Future work still needed for stateful API fuzzing, minimal method sets for library-wide coverage, and property-based security harnesses

## 7. Dataset & Metric
- **Libraries:** commons-cli (5K+ dependents), gson (27K+), guava (42K+), jackson-databind (36K+), jsoup (4K+), antlr4 (1K+)
- **Baselines:** OSS-Fuzz manually written harnesses, Jazzer AutoFuzz
- **Metrics:** Method-targeted line coverage, full target-scope (package) coverage, generation cost (tokens, USD, time), agent iterations and tool calls per phase, bug discovery count
- **Fuzzing setup:** 12-hour campaigns per target, single fuzzing thread, empty seed corpus, Jazzer fuzzer with JaCoCo instrumentation

## 8. Ket qua chinh
- Median 26% improvement in method-targeted coverage over OSS-Fuzz baselines across 5 main targets
- Outperform OSS-Fuzz by 6% and AutoFuzz by 5% on full package-scope coverage
- Generation cost: $1.34 (guava) to $6.25 (commons-cli), average $3.20; average time 599 seconds (~10 min)
- Discovered 3 unique previously unreported bugs: 2 null pointer exceptions in commons-cli option parsing, 1 index-out-of-bounds in jsoup HTML tree building (potential DoS vector)
- Zero false positives in crash reporting -- harnesses correctly distinguish expected from unexpected exceptions
- Research agent provides sufficient information in ~3.1 iterations on average; most harnesses compile correctly after initial synthesis with at most 6 patching iterations

## 10. Lien quan den de tai
Highly relevant to "Coding Agent Harness & Evaluation" -- this paper directly addresses automated harness generation using coding agents. The multi-agent architecture with specialized roles (research, generation, compilation repair, coverage analysis, refinement) demonstrates a concrete harness generation pipeline. The use of MCP for tool-augmented agent reasoning, method-targeted coverage as an evaluation metric, and agent-guided termination decisions are directly applicable patterns for coding agent evaluation frameworks. The coverage-guided refinement loop exemplifies how agents can iteratively improve generated code artifacts using feedback signals.
