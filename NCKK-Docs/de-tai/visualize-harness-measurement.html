HarnessEval: Cach Do 3 Chieu Harness

Visualize cach fork SWE-Agent thanh module + dat "dau do" (probe) de do tung chieu
Khong phai "inject code vao LLM" — ma la boc LLM trong cac lop config khac nhau va do output

0 Cau hoi cot loi: "Inject code" hay gi?

Tra loi ngan
Khong inject code vao LLM. Chung ta thay doi moi truong xung quanh LLM (tools nao co san, context duoc cat nhu nao, LLM nao duoc dung), roi do hanh vi cua ca he thong qua log files.

Giong nhu: ban khong mo nao bo ra de do — ban thay doi phong thi (de bai, thoi gian, dung cu) roi do ket qua bai thi.
Van de "Harness gan lien voi LLM"
Dung — harness va LLM la vong lap kin: harness cung cap tools → LLM chon tool → harness dispatch → ket qua ve LLM → ...
Chung ta KHONG the tach roi 100%. Nhung chung ta co the:
  • Do main effect: Trung binh qua tat ca LLM, harness config X co hieu qua hon config Y khong?
  • Do interaction: Harness config X tot hon voi Claude nhung xau voi DeepSeek?
  • Ca hai deu la ket qua khoa hoc co gia tri.

    5 PROBE POINTS — Noi chung ta "dat dau do":

    P1 CONTEXT PROBE     — Ghi lai: context gui cho LLM co bao nhieu token?
                            Thong tin quan trong co bi cat khong?
                            → Do Chieu 2: Context Utilization

    P2 TOOL LIST PROBE   — Ghi lai: LLM duoc cung cap nhung tool nao?
                            (Full: 12, Medium: 8, Minimal: 5)
                            → Anh huong Chieu 1: Tool Dispatch

    P3 BACKEND PROBE     — Ghi lai: LLM nao dang xu ly? Latency? Cost?
                            → Do Chieu 3: Backend Portability

    P4 TOOL CALL LOG     — Ghi lai: LLM chon tool nao? Arguments gi?
                            Co dung tool khong? Co thua khong?
                            → Do Chieu 1: Tool Dispatch (M1.1, M1.2, M1.3)

    P5 RESULT LOG        — Ghi lai: Task resolve thanh cong?
                            Bao nhieu turns? Bao nhieu tokens?
                            → Do Chieu 2 + input cho ANOVA
Khong phai "inject" — ma la "instrument" (dat dau do)
SWE-Agent da ghi log san tat ca tool calls va ket qua vao file JSON (trajectory files).
Chung ta chi can:
  1. Thay doi config (tool set, context strategy, backend) truoc khi chay
  2. Doc trajectory log sau khi chay
  3. Tinh metrics tu log data
Giong nhu: dat camera ghi hinh phong thi, KHONG can mo dau thi sinh.

CHIEU 1 Tool Dispatch Efficiency — "LLM co chon dung tool khong?"

Cach do: Doc log → so sanh voi "acceptable tools"
    HARNESS THAY DOI GI:
    Config A: Full (12 tools)    →  LLM thay: [bash, python, read, write, edit,
                                               glob, grep, find, git_diff, git_log,
                                               git_show, test_runner]

    Config B: Medium (8 tools)   →  LLM thay: [bash, python, read, write, edit,
                                               grep, git_diff, test_runner]

    Config C: Minimal (5 tools)  →  LLM thay: [bash, read, write, edit, grep]

    LOG GHI LAI GI (trajectory.json):
    {
      "turn": 3,
      "action": "tool_call",
      "tool": "grep",                    ← LLM chon tool nay
      "args": {"pattern": "def parse", "file": "src/parser.py"},
      "output": "line 42: def parse(...)...",
      "output_used_in_turns": [4, 5]     ← Output duoc dung o turn 4, 5
    }

    METRICS TINH TU LOG:

    ┌─────────────────────────────────────────────────────────────────┐
    │  M1.1  correct_selection_rate                                   │
    │                                                                 │
    │  = (so tool calls chon tool HOP LE) / (tong so tool calls)     │
    │                                                                 │
    │  "Hop le" = tool nam trong tap acceptable_tools cho task do     │
    │  Vi du: Task "tim bug" → acceptable = {grep, read, git_diff}   │
    │         LLM goi grep  → DUNG  ✓                                │
    │         LLM goi write → SAI   ✗ (chua can sua, dang tim)       │
    ├─────────────────────────────────────────────────────────────────┤
    │  M1.2  redundant_call_rate                                      │
    │                                                                 │
    │  = (so tool calls ma output KHONG duoc dung trong 3 turn)      │
    │    / (tong so tool calls)                                       │
    │                                                                 │
    │  Vi du: LLM goi grep → output khong xuat hien trong turn       │
    │         4, 5, 6 → THUA  (LLM goi ma khong dung ket qua)       │
    ├─────────────────────────────────────────────────────────────────┤
    │  M1.3  utilization_breadth                                      │
    │                                                                 │
    │  = (so tool KHAC NHAU da dung) / (tong so tools CO SAN)        │
    │                                                                 │
    │  Vi du: Co 12 tools, dung 4 → breadth = 0.33                   │
    │         Co 5 tools, dung 4  → breadth = 0.80                   │
    └─────────────────────────────────────────────────────────────────┘

    SO SANH 3 CONFIGS (cung 1 task, cung 1 LLM):

    Task: django__django-16379 (fix URL resolver bug)

    Full (12 tools):    grep→read→edit→test   | M1.1=1.00  M1.2=0.00  M1.3=0.33
    Medium (8 tools):   grep→read→edit→test   | M1.1=1.00  M1.2=0.00  M1.3=0.50
    Minimal (5 tools):  bash→read→edit→bash   | M1.1=0.75  M1.2=0.25  M1.3=0.60
                        ↑ bash thay grep (cham hon, co the sai)
                        ↑ 1 bash goi thua (khong dung output)

CHIEU 3 Backend Portability — "Harness co hoat dong tot voi moi LLM?"

Cach do: Cung 1 harness config, doi LLM, so sanh resolve rate
    GIU NGUYEN HARNESS, DOI LLM:

    Condition:  Full tools + Full history + [Claude]  → Resolve 45/150 = 30.0%
    Condition:  Full tools + Full history + [GPT-4o]  → Resolve 40/150 = 26.7%
    Condition:  Full tools + Full history + [DeepSeek] → Resolve 25/150 = 16.7%

    METRICS:

    ┌─────────────────────────────────────────────────────────────────┐
    │  M3.1  cross_backend_stddev                                     │
    │                                                                 │
    │  = std([30.0, 26.7, 16.7]) = 5.66                              │
    │                                                                 │
    │  Cang THAP → harness hoat dong DONG DEU tren moi LLM           │
    │  Cang CAO  → harness "thien vi" 1 LLM cu the                   │
    ├─────────────────────────────────────────────────────────────────┤
    │  M3.2  min_max_ratio                                            │
    │                                                                 │
    │  = min(RR) / max(RR) = 16.7 / 30.0 = 0.56                      │
    │                                                                 │
    │  Cang GAN 1.0 → portability tot (moi LLM gan nhu nhau)         │
    │  Cang GAN 0.0 → harness chi tot cho 1 LLM                      │
    └─────────────────────────────────────────────────────────────────┘


    VI SAO CHIEU NAY QUAN TRONG:

    Neu harness THAT SU "doc lap voi LLM":
    ┌─────────────────────────────────────────────────┐
    │  Harness tot   + Claude  = 70%                   │
    │  Harness tot   + GPT     = 68%                   │
    │  Harness tot   + DeepSeek = 65%                  │  ← stddev THAP
    │                                                   │
    │  Harness xau   + Claude  = 40%                   │
    │  Harness xau   + GPT     = 35%                   │
    │  Harness xau   + DeepSeek = 20%                  │  ← stddev CAO
    └─────────────────────────────────────────────────┘
    → Harness tot giup MOI LLM, khong chi LLM manh
    → Harness xau lam LLM yeu bi anh huong NHIEU HON
    → Day la interaction effect (H4 trong de cuong)

5 Buc tranh lon: Tai sao thiet ke nay hoat dong DU harness-LLM gan chat?

Analogy: Do anh huong cua "truong hoc" vs. "hoc sinh"
    VAN DE TUONG TU: Truong hoc tot co giup hoc sinh hoc gioi hon?

    "Truong" va "hoc sinh" KHONG doc lap:
    - Thay tot day bai kho hon → hoc sinh gioi vuot qua, hoc sinh yeu that bai
    - Hoc sinh gioi chon truong tot → selection bias
    - Lop hoc nho giup hoc sinh yeu NHIEU HON hoc sinh gioi

    Nhung ta VAN DO DUOC bang cach:
    ┌────────────────────────────────────────────────────────────────────────┐
    │                                                                        │
    │  Thay doi TRUONG (giu nguyen HOC SINH):                                │
    │    • Cung nhom hoc sinh → truong A vs B vs C                           │
    │    • Do diem thi                                                       │
    │    • ANOVA: Truong giai thich bao nhieu % variance cua diem?           │
