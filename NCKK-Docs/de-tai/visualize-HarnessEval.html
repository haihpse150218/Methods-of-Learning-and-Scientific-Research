3. Harness gom nhung gi? (Kien truc 5 lop)

Layer 5: ORCHESTRATION (Dieu phoi)
Agent loop: ReAct, Plan-Execute | Quyet dinh khi nao dung, khi nao tiep
Layer 4: SAFETY & PERMISSION (Bao mat)
Chan lenh nguy hiem (rm -rf) | Chong prompt injection | Sandbox
Layer 3: CONTEXT & MEMORY (Ngu canh)
Quan ly context window | Nen thong tin | Nho qua nhieu phien
Layer 2: TOOL SYSTEM (Cong cu)
Read, Write, Edit, Bash, Grep, Glob, WebSearch... | Goi tool nao? Khi nao?
Layer 1: MODEL BACKEND (Mo hinh)
Claude Sonnet | GPT-4o | DeepSeek-V3 | Qwen... (doi duoc!)
Diem mau chot: Layer 2-5 la HARNESS (chung ta danh gia cai nay).
Layer 1 la LLM (da co SWE-bench danh gia roi).

Vi du thuc te: Claude Code co ~40 tools, 5 lop safety, Adaptive Context Compaction. SWE-Agent chi co ~5 tools, 1 lop safety don gian. → Cung dung Claude Sonnet nhung ket qua rat khac!

4. Do gi? 5 Chieu danh gia Harness

Moi chieu tra loi 1 cau hoi cu the ve chat luong harness

D1: Tool Dispatch Efficiency
"Harness goi dung tool khong? Co bi thua khong?"
M1.1: Correct Selection Rate = % tool calls chon dung tool
M1.2: Redundant Call Rate = % tool calls khong can thiet

Vi du: Agent can doc file → goi Read (dung) hay goi Bash cat (sai tool)?
Agent goi Grep 5 lan cho cung 1 tu → 4 lan la thua.

D2: Context Utilization
"Harness nen context tot khong? Co mat thong tin quan trong?"
M2.1: Info Retention = % thong tin con lai sau khi nen
M2.2: Token Waste Ratio = % tokens khong lien quan

Vi du: Context 100K tokens nhung chi 20K lien quan → 80% waste.
Sau khi nen con 30K tokens nhung mat thong tin ve file quan trong → Info Retention thap.

D3: Safety Enforcement
"Harness chan duoc lenh nguy hiem khong?"
M3.1: Destructive Block Rate = % lenh nguy hiem bi chan
M3.2: Injection Resist Rate = % prompt injection bi phat hien

Vi du: Agent bi trick goi rm -rf / → harness chan duoc?
File .cursorrules chua lenh doc hai → harness phat hien?

D4: Session Continuity
"Harness co suy giam theo thoi gian khong?"
M4.1: Performance Decay = % giam hieu suat sau 100+ turns
M4.2: Instruction Adherence = % van nho system prompt

Vi du: Turn 10: agent lam dung yeu cau. Turn 150: agent "quen" yeu cau ban dau.
Day goi la "instruction fade-out" — harness tot se chen reminders de chong.

D5: Backend Portability
"Doi model khac thi harness con tot khong?"
M5.1: Cross-Backend Variance = do lech chuan giua cac model
M5.2: Min/Max Ratio = model yeu nhat / model manh nhat

Vi du: Harness A: Claude=70%, GPT=65%, DeepSeek=60% → variance thap = tot
Harness B: Claude=80%, GPT=30%, DeepSeek=20% → variance cao = harness chi tot cho Claude

5. Lam gi? Ablation Study — Tat/Bat tung component

Giong nhu thi nghiem y hoc: tat tung thanh phan de xem no dong gop bao nhieu

Condition Tool System Context Mgmt Safety Layer Memory Resolve Rate Nhan xet

Ket luan mau:
- Tool system dong gop nhieu nhat (~17%)
- Context management dong gop ~13%
- Memory dong gop ~7%
- Safety giam ~3% performance (trade-off)
- Harness tot giu Portability: Claude 65% → GPT 58% → DeepSeek 52% (variance thap)

8. Tai sao nghien cuu nay quan trong?

Cho Developer

Biet chon harness nao phu hop cho du an — khong chi chon model. "Du an can safety → chon OpenHands. Can portability → chon SWE-Agent."

Cho Researcher

Co framework chuan de so sanh harness moi. Thay vi chi bao "harness cua toi dat 70% tren SWE-bench," gio phai bao ro 5 chieu.

Cho Cong nghiep

Anthropic, OpenAI, Google se biet CHINH XAC component nao can cai thien trong harness cua ho, thay vi chi "thu dung model lon hon."

Khac biet 2: ABLATION — do dong gop tung component

Day la dong gop lon nhat. Chua ai lam trong coding.

Cau hoi chua ai tra loi duoc:
"Trong 1 harness, tool system dong gop bao nhieu %?
Context management dong gop bao nhieu %?
Neu bo safety layer, performance tang hay giam?"

Cach lam: TAT tung component, do su thay doi
Full harness:         65% resolve rate
Tat tool (5 tools):   48% → Tool dong gop ~17%
Tat context mgmt:    52% → Context dong gop ~13%
Tat safety:          68% → Safety GIAM 3% performance (trade-off!)

→ KET LUAN: "Dau tu tool system truoc, context thu hai,
  safety chap nhan giam nhe performance"

ToolLLM do tool accuracy nhung khong biet tool quan trong bao nhieu SO VOI context, safety. HarnessEval do TAT CA cung luc va so sanh.

Khac biet 3: CROSS-BACKEND — cung harness, doi LLM

CHUA CO PAPER NAO lam cross-backend comparison cho harness.

Cach lam: Giu CUNG harness, doi model
SWE-Agent + Claude:    65%
SWE-Agent + GPT-4o:    58%
SWE-Agent + DeepSeek: 52%
→ Variance = 6.5% → Portable (harness on dinh)

Claude Code + Claude:   70%
Claude Code + GPT-4o:   ??? (khong chay duoc!)
→ Portability = 0 (chi chay voi 1 model)

Khac biet 4: FRAMEWORK THONG NHAT (thay nhieu papers rieng le)

TRUOC HarnessEval:
Muon biet tool tot khong? → Doc paper ToolLLM
