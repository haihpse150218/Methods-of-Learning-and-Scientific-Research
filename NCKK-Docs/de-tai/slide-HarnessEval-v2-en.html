1. Problem: The Evaluation Paradox

The performance of a coding agent depends on two factors:

A. Base LLM
Claude, GPT, DeepSeek, Qwen...
Many benchmarks already evaluate this
B. Harness / Scaffold
Tool system, context manager, orchestration...
"The operating system of the agent" [A1]
Evidence: Lou et al. (2026) [A2] show a small model with a good harness can beat a large model with a weak harness, reducing invalid actions by 100%. Bui et al. (2026) [A1]: Adaptive Context Compaction cuts token usage by 54%.
The paradox: When we report "Agent A scored 70%," we do not know which harness component contributed to that result.

2. Research Gap: Three Genuine Reasons It Exists

We analyzed 49 papers (2023–2026): no work systematically evaluates the harness itself [S5].

1. Role mismatch
Princeton (SWE-Agent), Anthropic (Claude Code) are developers — incentivized to ship new harnesses, not to compare systems as evaluators.
Analogy: car makers vs Euro NCAP / IIHS
2. Field moving too fast
2023–2026: from a handful of harnesses to dozens. Nobody has paused to meta-evaluate while still racing to build.
3. Only indirect evidence
Lou et al. compared harnesses on game tasks [A2]. Robeyns et al. self-improved a harness but did not measure each component [B10].
Research gap: No framework exists to (a) measure the quality of individual harness components, (b) decompose harness vs base-LLM contribution, and (c) compare harnesses on unified quality dimensions.

4. Research Hypotheses

H1
Tool system has the largest effect size (Cohen's d > 0.5) among the three harness components on resolve rate.
H2
Context management has a medium effect size (Cohen's d 0.3 – 0.5).
H3
Harness configuration explains at least 20% of resolve-rate variance (eta² ≥ 0.20), independent of the LLM backend.
H4
Variance across LLM backends decreases when harness quality is higher (statistically significant interaction effect).
Neutral framing: No specific numbers predicted ("30–40%"). Only testable directions → avoid confirmation bias. Supporting or refuting a hypothesis is equally valuable.

9. Two-Way ANOVA — Decomposing Harness vs LLM

"Independent of the LLM" does not mean harness and LLM are uncorrelated. It means we can decompose the variance due to harness vs the variance due to LLM.

Statistical model:

Total Variance = Var(Harness) + Var(LLM) + Var(Harness x LLM) + Error

Factor A: Harness config   (9 levels: 3 tools x 3 contexts)
Factor B: LLM backend      (3 levels: Claude, GPT, DeepSeek)
DV:        Resolve rate

If Var(Harness) is significant    --> harness has an INDEPENDENT effect
If Var(H x LLM) is significant    --> there is an INTERACTION
eta-squared                       --> % of variance explained per factor
Main theoretical contribution: Formalizing how to separate harness quality from LLM quality. Post-hoc: Tukey HSD pairwise, Cohen's d effect sizes, Bonferroni correction for 7 metrics.
