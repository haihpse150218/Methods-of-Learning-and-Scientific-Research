Tai sao can Modular Harness?

Tu van de cua SWE-Agent goc → Ky thuat tach module → Tai sao day la DIEU KIEN TIEN QUYET cho nghien cuu

1

Van de: SWE-Agent goc la "khoi bet-tong"

Phep an du
SWE-Agent goc giong nhu 1 chiec xe hoi nguyen khoi.
Ban muon biet: "Dong co dong gop bao nhieu % vao toc do? Lop dong gop bao nhieu %?"
Nhung dong co, hop so, lop deu HAN CUNG — khong the thao ra doi rieng tung mon. 
SWE-AGENT GOC — MONOLITHIC

  ┌──────────────────────────────────────────────────────────────────────────┐
  │                                                                          │
  │   config.yaml  (MOT FILE DUY NHAT)                                      │
  │   ┌────────────────────────────────────────────────────────────────┐     │
  │   │                                                                │     │
  │   │   model: "gpt-4o"                    ← Backend dinh cung      │     │
  │   │   tools: [bash, edit, read, ...]     ← Tools dinh cung        │     │
  │   │   max_context: 100000               ← Context dinh cung       │     │
  │   │   prompt_template: "..."            ← Tat ca trong 1 file     │     │
  │   │   safety_settings: ...                                         │     │
  │   │   orchestration: ...                                           │     │
  │   │                                                                │     │
  │   └────────────────────────────────────────────────────────────────┘     │
  │                                                                          │
  │   swe_agent/                                                             │
  │   ├── agent.py         ← Tool logic + Context logic + LLM call          │
  │   │                       TRON LAN trong cung 1 class                │
  │   ├── tools.py         ← Hardcode danh sach tools                       │
  │   ├── environment.py   ← Docker + execution                             │
  │   └── models.py        ← API calls (nhung PHU THUOC vao agent.py)      │
  │                                                                          │
  │   VAN DE:                                                              │
  │   • Muon doi tools? → Phai sua agent.py (co the hong context logic)     │
  │   • Muon doi context? → Phai sua agent.py (co the hong tool logic)      │
  │   • Muon doi LLM? → Phai sua models.py + agent.py + config.yaml         │
  │   • Moi thay doi → KHONG BIET co anh huong gi khac khong                │
  │                                                                          │
  └──────────────────────────────────────────────────────────────────────────┘
Hau qua khi dung monolithic cho nghien cuu
Khi ban sua code de "doi tool set tu 12 → 8", ban co the vo tinh:
  • Thay doi cach prompt duoc tao (vi prompt liet ke tools)
  • Lam hong error handling (vi 1 tool bi tat nhung code van reference no)
  • Doi thu tu xu ly (vi if/else chain thay doi)
→ Performance thay doi, nhung khong biet vi tool hay vi code bi hong
→ Ket qua thuc nghiem vo gia tri

Giai phap: Tach thanh 3 module hoan doi

Phep an du
Modular harness giong nhu dan am thanh voi 3 o cam doc lap.
O cam 1: Micro (tool set) — cam bat ky micro nao vao
O cam 2: Mixer (context strategy) — cam bat ky mixer nao vao
O cam 3: Loa (LLM backend) — cam bat ky loa nao vao
Doi micro KHONG anh huong gi den loa. Do am thanh → biet micro dong gop bao nhieu.

So sanh truc tiep: Doi 1 bien voi Monolithic vs. Modular

Gia su ban muon doi tool set tu Full(12) → Minimal(5), giu nguyen context va backend. Day la qua trinh voi tung loai kien truc:

MONOLITHIC (SWE-Agent goc)

Buoc 1: Mo config.yaml
  Sua tools: [bash, ..., test_runner]
  → Xoa 7 tools thu cong

Buoc 2: Mo agent.py
  Tim tat ca cho tham chieu tools da xoa
  Vi du: if "git_log" in tools: ...
  → Comment out? Xoa? Doi logic?
  → CO THE HONG LOGIC KHAC

Buoc 3: Mo prompt_template
  Prompt liet ke 12 tools cho LLM
  → Phai sua prompt (xoa 7 tools)
  → PROMPT THAY DOI = BIEN KHAC
     THAY DOI (khong chi tool set)

Buoc 4: Chay test → co the FAIL
  Vi code van reference tools da xoa
  → Phai sua them code
  → SUA THEM = THEM BIEN THAY DOI

Buoc 5: Lam lai buoc 1-4
  cho MOI to hop khac
  → 27 lan × 4 buoc = 108 lan sua code
  → 108 co hoi gay bug

TONG:
  Thay doi: tool set + code logic
           + prompt + error handling
  So bien thay doi: 4+ (khong kiem soat)
  Ket qua: KHONG TIN DUOC
VS

MODULAR (Harness fork)

Buoc 1: Doi 1 dong trong YAML:

  # truoc:
  tool_provider: "full"

  # sau:
  tool_provider: "minimal"

  → XONG. Chi 1 dong.




He thong tu dong:
  Factory doc "minimal"
  → Tao MinimalToolProvider
  → Provider tra ve 5 tools
  → SWE-Agent chi thay 5 tools
  → Prompt TU DONG chi liet ke 5 tools
  → CODE KHONG BI SUA




27 to hop?
  generate_all_configs()
  → 27 HarnessConfig objects
  → 27 YAML files tu dong
  → 0 lan sua code



TONG:
  Thay doi: CHI tool set
  Moi thu khac: GIONG HET
  So bien thay doi: DUNG 1
  Ket qua: TIN DUOC

Dam bao an toan: Equivalence Test (+-3%)

Rui ro lon nhat cua fork
Khi tach module, co the vo tinh lam hong SWE-Agent.
Neu fork bi hong → 7,050 evaluations chay tren harness hong → toan bo data vo nghia.

  TAI SAO 3% (khong phai 0%)?

  LLM co tinh NGAU NHIEN (du temperature=0 van co variance nho).
  Cung config, chay 2 lan co the chenh lech 1-2%.
  Nen 3% la nguong chap nhan duoc — chenh lech do nhieu, khong do fork hong.

Tom tat 1 cau
Modular harness khong phai "thiet ke dep" — no la PHUONG PHAP NGHIEN CUU. Khong co no thi khong the lam ablation study, khong co ANOVA, khong co de tai. No giong nhu ong nghiem trong phong thi nghiem — khong co thi khong lam thi nghiem duoc.
