1 The Problem: Monolithic Harness

BEFORE — Original SWE-Agent

Everything is hardcoded

def solve(task):
    # Tools: fixed, cannot be changed
    tools = [edit, terminal, git, search]

    # Context: fixed, cannot be changed
    context = get_full_history()

    # LLM: fixed, cannot be changed
    response = openai.chat(context)

    # => Changing 1 part = rewriting EVERYTHING
The Problem
All 3 components are tightly coupled via direct function calls.
Cannot change one while keeping the other two constant.
→ Ablation experiments are impossible.
→
AFTER — Modular Fork

Dependency Injection

def solve(task, tool_provider,
              context_strategy,
              llm_backend):

    # Tools: swappable
    tools = tool_provider.get_tools()

    # Context: swappable
    context = context_strategy.build(history)

    # LLM: swappable
    response = llm_backend.generate(context)

    # => Change 1 part, keep the other 2
The Solution
Use Interfaces to decouple each layer.
Each layer has multiple implementations, selected via YAML config.
→ Techniques: Strategy Pattern + Dependency Injection

2 Modular Harness Architecture — 3 Swappable Layers

🔧 LAYER 1: Tool Dispatch Layer interface: ToolProvider
Config A: Full
file_edit + terminal + git + search
Complete toolset matching original SWE-Agent
Config B: Minimal
file_edit only
Agent must do everything with a single tool
Config C: No-Git
file_edit + terminal + search
No git — measures the role of version control
▼
📝 LAYER 2: Context Management Layer interface: ContextStrategy
Strategy 1: Full History
Retains the entire conversation history
Best information quality, highest token cost
Strategy 2: Sliding Window
Keeps only the last 10 steps
Saves tokens, loses distant context
Strategy 3: ACC
Adaptive Context Compaction
Keeps important info, compresses the rest
▼
🧠 LAYER 3: LLM Backend Layer interface: LLMBackend
Backend I: GPT-4o
OpenAI — widely-used commercial model
Backend II: Claude Sonnet
Anthropic — strong at code reasoning
Backend III: DeepSeek
Open-source — cost-effective, high performance

Similar to a neuroscientist deactivating specific brain regions to see what each one controls.

6 Key Techniques: Dependency Injection Explained

Strategy Pattern — Multiple implementations per layer

# Interface (contract)
class ContextStrategy(ABC):
    def build(self, history) -> str:
        ...  # each subclass implements this

# Implementation 1: Keep everything
class FullHistoryStrategy(ContextStrategy):
    def build(self, history):
        return "\n".join(history)  # keep all

# Implementation 2: Keep last 10 steps
class WindowStrategy(ContextStrategy):
    def build(self, history):
        return "\n".join(history[-10:])  # last 10 steps

# Implementation 3: Adaptive compression
class ACCStrategy(ContextStrategy):
    def build(self, history):
        return self.compress(history)  # smart compression

Factory Pattern — Select implementation from config

# Read YAML config → create correct object
def create_experiment(config):
    tool = ToolFactory.create(config["tool_provider"])
    ctx  = ContextFactory.create(config["context_strategy"])
    llm  = LLMFactory.create(config["llm_backend"])
    return ModularHarness(tool, ctx, llm)

# Usage
harness = create_experiment({
    "tool_provider": "minimal",
    "context_strategy": "acc",
    "llm_backend": "claude-sonnet"
})
result = harness.solve(task)  # run!
